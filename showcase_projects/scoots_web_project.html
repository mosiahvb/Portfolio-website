Mock Rental Car Website

Modern Web Design Meets Real-World Use Case

What It Is

This is a multi-page mock rental car website designed to simulate a real-world car rental experience. It includes a homepage, rentals listing, reservation form, and contact information. Users can browse available cars, check pricing, view pickup locations on a map, and even get real-time weather conditions from the OpenWeatherMap API. The website presents a polished UI experience while working entirely with mock data.

Why I Built It

I created this website to practice integrating multiple front-end skills into a cohesive project. The idea of building a real-world business website — one that could serve a customer and help them reserve a vehicle — felt like a great way to build confidence with responsive design, APIs, and JavaScript-driven content. I also wanted to experiment with real-time data (like weather) to make the project more dynamic and engaging.

How I Built It

This site uses HTML for structure, CSS for styling, and JavaScript to dynamically load vehicle data from local JSON files. Car details like type, features, price, and image are inserted into the DOM with JavaScript on the Rentals page. The Contact and Reservation pages include forms and embedded tools for location and communication.

I used the OpenWeatherMap API to fetch current weather information for the business’s main location, which updates automatically on page load. A custom Google Map embed displays two fictional rental office locations.

  • Home Page: Welcome banner, brief intro, weather widget.
  • Rentals Page: Dynamically loaded car listings with pricing.
  • Reservations Page: Simple form with date, time, and vehicle options.
  • Contact Page: Location map and contact details.

Challenges I Faced

One of the more difficult parts of this project was working with asynchronous JavaScript. Pulling weather data from the OpenWeatherMap API and handling edge cases like connection errors required trial and error.

Other challenges and takeaways:

  • Dynamic DOM Insertion: I learned how to iterate over JSON and cleanly insert cards into the DOM using templates.
  • Mobile Responsiveness: Making the Rentals layout look good on mobile required lots of CSS tweaking and media queries.
  • Form Validation: I added basic HTML validation but realized I’d like to add more logic-driven form handling in the future.

Tools & Tech Used

  • HTML
  • CSS
  • JavaScript
  • JSON
  • OpenWeatherMap API
  • Google Maps Embed

What I'd Do Differently

As with all projects, I found several things I would improve next time:

  • 1: Add filtering and sorting features for rentals (e.g., by price or car size).
  • 2: Integrate a backend service like Firebase to support live data instead of local JSON.
  • 3: Build a more advanced weather widget that responds to user-selected branch locations.
  • 4: Improve accessibility features such as ARIA labels and keyboard navigation.

This project helped me practice combining UI, data, and responsiveness into a single experience. It was a great step in applying design thinking to web development.
