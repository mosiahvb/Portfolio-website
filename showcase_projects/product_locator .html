Inventory Location Web App

A Flask-based internal web application used at Pro Moto Billet to streamline product location for packaging and shipping teams.

What It Is

This web application allows warehouse and shipping personnel to:

  • Search for part numbers and instantly retrieve their storage location(s)
  • View location data including aisle, section, and shelf
  • See multiple storage locations for a single item
  • Update, move, or remove part location information
  • Maintain an up-to-date product location database using a clean and minimal interface

It was built to solve a real problem: scattered storage and reliance on tribal knowledge that led to delays in order fulfillment.

Why I Built It

I created this web app to:

  • Eliminate bottlenecks in our packaging and shipping workflow
  • Enable faster product location for employees with minimal training
  • Improve inventory accuracy and speed up shipping processes
  • Learn how to build and deploy a full-stack app using Flask, SQL, and JavaScript
  • Deliver a production-ready internal tool for daily business operations

This app transformed our internal process. New hires can now locate parts without assistance, saving valuable time.

How I Built It

Technologies and frameworks used:

  • Python + Flask for backend server and routing
  • SQLite for the relational database
  • HTML, CSS, JavaScript for the frontend UI

Workflow Overview:

  1. Staff opens the web app in a browser
  2. Searches for a part number
  3. The app queries the SQLite database
  4. Location info (aisle, section, shelf) is returned
  5. Users can add, update, or remove entries as needed
  6. Changes are reflected in real time

Challenges I Faced

  • Database integration: Learning how to connect SQLite with Flask and handle CRUD operations
  • Frontend/backend communication: Syncing Python, HTML/CSS, and JavaScript smoothly
  • User-friendliness: Designing an intuitive interface for non-technical staff
  • Data integrity: Ensuring accurate, consistent location data
  • Real-world testing: Validating the tool in a live warehouse environment

Tools & Tech Used

  • Python
  • Flask
  • SQLite
  • HTML
  • CSS
  • JavaScript

What I'd Do Differently

  • Add product images for visual reference
  • Enable notes/tags for special storage instructions
  • Improve search with filters or fuzzy search
  • Implement user authentication for secure updates
  • Deploy to the cloud for access across multiple departments
