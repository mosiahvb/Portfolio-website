First Aid

Empower Yourself With Simple, Life-Saving Skills

What It Is

This web application is designed to help users learn the fundamentals of first aid through a structured, three-level learning path. Each level covers progressively more advanced life-saving topics, from everyday cuts and scrapes to trauma response techniques like CPR and bleeding control. Users can explore pre-set learning paths or curate a custom playlist of videos to study at their own pace.

Why I Built It

I was raised in a household of first responders and medical professionals — including my great-grandfather, grandmother, mother, and father — who passed down real-world medical knowledge and the importance of being prepared to help others. I created this website to make basic, life-saving first aid skills accessible to those who didn’t grow up with the same training. It's alarming that roughly 31,000 people die each year from preventable blood loss after car accidents — often because no one nearby knows what to do. This site is my way of sharing that critical knowledge.

How I Built It

This site was built with a minimal HTML structure — almost everything you see on the page is dynamically generated using JavaScript. The JavaScript handles the heavy lifting: it loads data from local JSON files, organizes it, formats the content, and injects it into the HTML. Aside from the basic layout like the navigation and header, all of the educational content, video embeds, and user interactions are managed through JavaScript. The design is styled with CSS, but the core functionality is powered by JavaScript.

This site is structured around a three-level learning system:

  • Level 1: Everyday first aid (cuts, scrapes, burns).
  • Level 2: Serious incidents (CPR, fractures).
  • Level 3: Trauma response (bleeding control, emergency care).

Content is loaded from local JSON files, and instructional videos are embedded from YouTube. I used JavaScript and browser localStorage to let users build a custom "Watch List" playlist. Users can mark videos as completed, which removes them from their list and helps track learning progress.

Challenges I Faced

I started this project with a simple design sketch to guide the development process. My focus was on building the core functionality first, then refining the user experience through iteration.

Key lessons I learned during the process include:

  • Modular JavaScript: I struggled to consolidate multiple functions into reusable modules, especially when working with asynchronous JSON data. This is an area I plan to revisit and improve.
  • Video Performance Optimization: Embedding YouTube videos directly slowed down page load times. I’m researching lazy loading techniques and thumbnail previews to improve performance.
  • Iterative Learning: Testing in small steps helped me identify problems early. Each feature taught me to think like both a developer and a designer.

What I'd Do Differently

As I reflect on this project, there are several things I would approach differently if I were starting over:

  • 1: Plan for modular code earlier to make JavaScript functions more reusable and easier to maintain.
  • 2: Structure JSON and video content with scalability in mind — allowing future updates to be more seamless.
  • 3: Prioritize responsive design from the start to avoid reworking layout for mobile devices later.
  • 4: Implement lazy loading and thumbnail previews sooner to improve video performance and loading times.
  • 5: Focus more on accessibility, including keyboard navigation and clear contrast for users with visual impairments.

This project has been a meaningful learning experience, and I’m proud of what I’ve accomplished. Going forward, I plan to build on this foundation with better practices, more features, and a sharper focus on user needs.
