Port Scanner

A Python-based CLI tool for scanning open ports and vulnerabilities using Nmap and the NVD API

What It Is

This is a command-line Python tool that performs network reconnaissance on a given domain or IP address. It resolves domain names, scans open ports with nmap, identifies service types and versions, and queries the National Vulnerability Database (NVD) to find related CVEs (Common Vulnerabilities and Exposures). It outputs all results to the terminal, helping users analyze and understand potential risks.

The tool is intended for internal network auditing and educational use, giving users a better understanding of how attackers assess vulnerabilities.

How I Built It

The project uses several Python libraries and system tools to perform its operations:

  • Python for the overall logic and automation
  • Socket for DNS resolution and basic IP handling
  • python-nmap to interface with the Nmap network scanning tool
  • Requests to make HTTP calls to the National Vulnerability Database (NVD)
  • OS to securely access environment variables such as the NVD API key stored in a .env file

Workflow:

  1. User enters a domain or IP address
  2. Tool resolves the domain to an IP (if needed)
  3. nmap scans the IP for open ports
  4. Service names and versions are extracted
  5. Queries are made to the NVD API for known vulnerabilities
  6. Results are displayed to the terminal

Challenges I Faced

  • Choosing a scanning method: I initially considered writing my own port scanner but opted for nmap due to its performance and reliability.
  • Parsing output: Making sense of nmap output required carefully handling and interpreting its structured results.
  • Working with the NVD API: Matching port information with CVEs wasn't always straightforward due to inconsistent naming and API rate limits.
  • Securing the API key: I used a .env file and Python's os module to securely store and retrieve the key.
